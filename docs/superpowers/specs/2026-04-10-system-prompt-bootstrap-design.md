# System Prompt + Structured Bootstrap Design

## Problem

`--agent` does NOT fully replace Claude Code's default system prompt — it adds ~8600 tokens of CC infrastructure on top. Bootstrap instructions from BOOTSTRAP.md (~900 tokens) are ~10% of the total system prompt, causing the model to default to CC behavior ("Hi! How can I help?") instead of running the onboarding flow.

`--system-prompt-file` replaces the CC prompt entirely. Tested and confirmed working for bootstrap.

## Solution

Two-layer prompting for every `claude -p` invocation:

1. **`--system-prompt-file system-prompt.md`** — base context: "who you are" (same for bootstrap and normal)
2. **`--agent {name}[-bootstrap]`** — behavior: "what to do" (agent-specific instructions via @ file references)

Plus `--json-schema` always present (bootstrap-schema or reply-schema).

## System Prompt File

**Generated by:** `generate_system_prompt()` in `codegen/agent_def.rs`
**Written to:** `agent_dir/.claude/system-prompt.md`
**Synced to sandbox:** `/sandbox/.claude/system-prompt.md`

### Content (~200 words)

```
You are {agent_name}, a RightClaw agent.

RightClaw is a multi-agent runtime for Claude Code built on NVIDIA OpenShell.
Each agent runs as an independent Claude Code session inside its own sandbox
with declarative YAML policies. Agents have persistent memory, scheduled tasks
(cron), and tool management via MCP.

Source: https://github.com/onsails/rightclaw

## Environment

- Sandbox: {sandbox_desc}
- Agent name: {agent_name}

## MCP

You are connected to the `right` MCP server for persistent memory, cron job
management, and external MCP server management. Run `mcp_list` to see all
configured servers.
```

Parameters interpolated by codegen: `agent_name`, `sandbox_desc` (from SandboxMode).

Does NOT include: agent behavior, identity files, bootstrap instructions — those are in the agent definition.

## Worker Bootstrap Flow

### Invocation

```
claude -p \
  --system-prompt-file /sandbox/.claude/system-prompt.md \
  --agent {name}-bootstrap \
  --json-schema <bootstrap-schema-json> \
  --dangerously-skip-permissions \
  --mcp-config /sandbox/mcp.json --strict-mcp-config \
  --output-format json
```

Normal mode: same but `--agent {name}` and reply-schema.

### Bootstrap Completion (server-side gated)

```
if bootstrap_complete == Some(true) && should_accept_bootstrap(agent_dir):
    → delete session, delete BOOTSTRAP.md, bootstrap done
if bootstrap_complete == Some(true) && !should_accept_bootstrap(agent_dir):
    → log warning, ignore claim, continue bootstrap mode
if bootstrap_complete != Some(true):
    → normal response, continue bootstrap mode
```

Reverse sync is BLOCKING in bootstrap mode (await before file check).
Fire-and-forget in normal mode (existing behavior).
