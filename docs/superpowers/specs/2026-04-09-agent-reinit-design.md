# Agent Re-initialization (`agent init --force`)

## Problem

During development iteration and agent retirement+replacement, there's no way to wipe an agent's state and start fresh. `init_agent()` rejects if the agent directory already exists. Sandboxes persist but have no CLI-driven cleanup. Users must manually delete directories and sandboxes.

## Solution

Extend `rightclaw agent init <name>` with `--force` flag that confirms, wipes all agent state (including OpenShell sandbox), and re-runs init. By default reuses the existing `agent.yaml` config to avoid re-answering wizard questions during dev iteration.

## CLI Interface

```
rightclaw agent init <name> [--force] [--fresh] [--yes]
```

| Flag | Description |
|------|-------------|
| `--force` | If agent exists: confirm, wipe (sandbox + agent dir), re-init |
| `--fresh` | With `--force`: re-run interactive wizard instead of reusing existing config. Error without `--force`. |
| `--yes` / `-y` | Skip confirmation prompt (scripting/CI) |

## Wipe Sequence

When `--force` is used and agent directory exists:

1. **Read existing config** ŌĆö parse `AgentConfig` from `agent.yaml`. Skip if `--fresh`.
2. **Confirmation prompt** (unless `--yes`):
   ```
   Agent "right" already exists at ~/.rightclaw/agents/right/
   This will permanently delete:
     - All agent files (identity, memory, skills, config)
     - OpenShell sandbox "rightclaw-right" (if exists)
   Continue? [y/N]
   ```
3. **Check agent not running** ŌĆö read runtime state. If agent is in process-compose, abort: `"Agent '<name>' is currently running. Run 'rightclaw down' first."`
4. **Delete sandbox** ŌĆö `delete_sandbox("rightclaw-<name>")`. Best-effort, log at debug level on failure.
5. **Delete SSH config** ŌĆö remove `~/.rightclaw/run/ssh/rightclaw-<name>.ssh-config` if exists.
6. **Delete agent directory** ŌĆö `rm -rf ~/.rightclaw/agents/<name>/`
7. **Run `init_agent()`** ŌĆö with saved config (default) or fresh wizard (`--fresh`).

## Config Preservation

When `--force` without `--fresh`, the following values are read from existing `agent.yaml` before deletion and passed back to `init_agent()`:

**Preserved:**
- `sandbox.mode` (openshell / none)
- `network_policy` (restrictive / permissive)
- `telegram_token`
- `allowed_chat_ids`
- `model`
- `env`

**Not preserved (regenerated):**
- `secret` ŌĆö regenerated by codegen pipeline on next `rightclaw up`
- `memory.db` ŌĆö wiped
- Identity files (IDENTITY.md, SOUL.md, USER.md) ŌĆö recreated by bootstrap
- Custom skills ŌĆö wiped
- `.claude.json`, `settings.json`, `mcp.json` ŌĆö regenerated

## InitOverrides Struct

```rust
struct InitOverrides {
    sandbox_mode: SandboxMode,
    network_policy: NetworkPolicy,
    telegram_token: Option<String>,
    allowed_chat_ids: Vec<i64>,
    model: Option<String>,
    env: Option<BTreeMap<String, String>>,
}
```

When `init_agent()` receives `Some(overrides)`, it skips the wizard and applies saved values to the template `agent.yaml`.

## Error Handling

| Case | Behavior |
|------|----------|
| Corrupted `agent.yaml` with `--force` (no `--fresh`) | Abort with: `"Could not read existing config: {error}. Use --fresh to reconfigure from scratch."` |
| Sandbox doesn't exist | Debug-level log, continue |
| Agent currently running | Abort: `"Agent '<name>' is currently running. Run 'rightclaw down' first."` |
| `--force` on nonexistent agent | No-op, run normal `init_agent()` |
| `--fresh` without `--force` | Clap validation error: `"--fresh requires --force"` |
