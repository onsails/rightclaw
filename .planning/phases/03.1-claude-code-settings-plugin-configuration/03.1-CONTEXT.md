# Phase 3.1: Claude Code Settings & Plugin Configuration - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Write `.claude/settings.json` in the agent directory during `rightclaw init` with `enabledPlugins` for Telegram when token is provided. This eliminates the manual `/plugin install` step. Minimal scope — just the settings file, no marketplace.

</domain>

<decisions>
## Implementation Decisions

### Settings File
- **D-01:** `rightclaw init` creates `.claude/settings.json` in the agent directory when Telegram token is provided
- **D-02:** Content: `{"enabledPlugins": {"telegram@claude-plugins-official": true}}`
- **D-03:** When no Telegram token: do NOT create `.claude/settings.json` (no need — no plugins to enable)
- **D-04:** User-level `~/.claude/settings.json` loads normally — it has auth, permissions, theme. No isolation needed.

### Claude's Discretion
- Whether to create `.claude/` directory even without Telegram (for future extensibility)
- Settings file formatting (pretty-print vs compact JSON)

</decisions>
