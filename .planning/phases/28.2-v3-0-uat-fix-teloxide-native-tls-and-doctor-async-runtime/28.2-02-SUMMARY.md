## Accomplishments

- Fixed nested tokio runtime panic in `fetch_webhook_url` — `rightclaw doctor` no longer panics when called from `async main`
- Replaced `Runtime::new().block_on()` with `block_in_place(|| Builder::new_current_thread().build()?.block_on())` — safe pattern for sync functions called from multi-thread async context

## Files Created/Modified

## Decisions Made

- Use `tokio::task::block_in_place` + `Builder::new_current_thread()` instead of `Runtime::new()` — `block_in_place` temporarily releases the current thread from the multi-thread pool, making it safe to create a new single-thread runtime inside
- Telegram API returns HTTP 200 with empty `result.url` for invalid tokens (not a network error) — regression test asserts no-panic by completing normally, not by asserting `is_err()`
