### Notable Deviation: native-tls → rustls

Plan 28.2-01 specified `"native-tls"` but the implementation used `"rustls"`. This is a correct deviation:
- `native-tls` triggers `openssl-sys` which requires OpenSSL headers via pkg-config; the Nix devenv does not expose them, causing a build failure (exit 101).
- `rustls` is pure-Rust, requires no system libraries, and is already the TLS backend for the workspace `reqwest` dependency — making the workspace consistently rustls-only.
- The semantic goal (teloxide has a working TLS backend) is fully achieved.

### Notable Deviation: regression test assertion

Plan 28.2-02 specified `assert!(result.is_err())` but the Telegram API returns HTTP 200 with `{"ok": false, "result": {"url": ""}}` for invalid tokens — `fetch_webhook_url` returns `Ok("")` in this case, not `Err`. The test was correctly adjusted to an implicit no-panic check (`let _result = fetch_webhook_url(...)`). The critical invariant — no panic in multi-thread async context — is still fully verified.
