  <action>
    1. Switch main to async: change `fn main()` to `#[tokio::main] async fn main()` (needed for PcClient async methods).

    2. Add five new subcommands to Commands enum:
       ```rust
       /// Launch agents with process-compose
       Up {
           /// Only launch specific agents (comma-separated)
           #[arg(long, value_delimiter = ',')]
           agents: Option<Vec<String>>,
           /// Launch in background with TUI server
           #[arg(short, long)]
           detach: bool,
           /// Run without OpenShell sandbox (development only)
           #[arg(long)]
           no_sandbox: bool,
       },
       /// Stop all agents and destroy sandboxes
       Down,
       /// Show running agent status
       Status,
       /// Restart a single agent
       Restart {
           /// Agent name to restart
           agent: String,
       },
       /// Attach to running process-compose TUI
       Attach,
       ```

    3. Implement Commands::Up handler:
       a. Call `verify_dependencies(no_sandbox)` -- fail fast if tools missing
       b. Resolve run_dir = home.join("run"), create it with `std::fs::create_dir_all`
       c. Check for stale socket: if run_dir.join("pc.sock") exists, try PcClient::new + health_check. If alive, error "rightclaw is already running. Use `rightclaw down` first or `rightclaw attach` to connect." If dead, delete stale socket file.
       d. Discover agents via `discover_agents(&home.join("agents"))`
       e. If --agents filter provided, filter discovered agents by name. Error if any requested agent not found (list available agents in error message).
       f. For each agent: generate_wrapper(agent, no_sandbox), write to run_dir.join(format!("{}.sh", agent.name)), set executable permission via `std::os::unix::fs::PermissionsExt` (mode 0o755)
       g. Generate process-compose.yaml via generate_process_compose(&agents, &run_dir), write to run_dir.join("process-compose.yaml")
       h. Write RuntimeState to run_dir.join("state.json") with agent names and sandbox names
       i. Build process-compose Command: args = ["up", "-f", config_path, "--unix-socket", socket_path]
       j. If detach: add "--detached-with-tui", spawn child, wait 500ms, health_check, print success message
       k. If foreground: use tokio::process::Command status().await, propagate exit code

    4. Implement Commands::Down handler:
       a. Read state from run_dir.join("state.json") via read_state. If no state file, error "No running instance found."
       b. Try PcClient shutdown (best-effort -- PC may already be dead)
       c. If no_sandbox was NOT used (check state or just always try): call destroy_sandboxes with agents from state -- per D-12, D-13
       d. Delete socket file if it exists
       e. Print "All agents stopped."
       NOTE: Per D-13, do NOT delete run/ files. They stay for debugging.

    5. Implement Commands::Status handler:
       a. Build PcClient from run_dir.join("pc.sock"). If socket doesn't exist, error "No running instance. Run `rightclaw up` first."
       b. Call list_processes(), format as table: name, status, PID, uptime
       c. Print table to stdout

    6. Implement Commands::Restart handler:
       a. Build PcClient from run_dir.join("pc.sock")
       b. Call restart_process(&agent)
       c. Print "Restarted agent: {agent}"

    7. Implement Commands::Attach handler (per D-16):
       a. Check socket_path exists, error if not
       b. Use `std::os::unix::process::CommandExt` to replace current process with process-compose:
          ```rust
          use std::os::unix::process::CommandExt;
          let err = std::process::Command::new("process-compose")
              .arg("attach")
              .arg("--unix-socket")
              .arg(&socket_path)
              .exec();
          return Err(miette::miette!("Failed to attach: {err}"));
          ```
       c. The .exec() replaces the process -- it only returns on error.

    NOTE on RuntimeState and no_sandbox: The `up` command needs to record whether --no-sandbox was used so `down` knows whether to attempt sandbox cleanup. Add a `no_sandbox: bool` field to RuntimeState struct in sandbox.rs (this is a minor addition to Plan 02's artifact -- add it here if not present).
