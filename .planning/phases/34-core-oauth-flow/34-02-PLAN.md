    Add to `crates/rightclaw/src/mcp/oauth.rs`:

    1. `pub async fn discover_as(client: &reqwest::Client, server_url: &str) -> Result<AsMetadata, OAuthError>`:
       - Parse server_url as URL, extract path component (strip trailing slash)
       - **Step 1 — RFC 9728 resource metadata:**
         URL has path → `{origin}/.well-known/oauth-protected-resource{path}`
         URL has no path → `{origin}/.well-known/oauth-protected-resource`
         Send GET. If 2xx → parse as `ResourceMetadata`, extract first `authorization_servers[0]` as AS URL. If 404 → Step 2. If 5xx → return `OAuthError::DiscoveryFailed("RFC 9728 server error: {status}")`.
       - **Step 2 — RFC 8414 AS metadata (using AS URL from step 1, or server_url as fallback):**
         AS URL has path `/foo` → try `{origin}/.well-known/oauth-authorization-server/foo`
         AS URL has no path → try `{origin}/.well-known/oauth-authorization-server`
         Send GET. If 2xx → parse as `AsMetadata`, return. If 404 → Step 3. If 5xx → return error.
       - **Step 3 — OIDC well-known:**
         Try `{as_origin}/.well-known/openid-configuration`
         If path exists, also try `{as_origin}/{path}/.well-known/openid-configuration`
         Send GET. If 2xx → parse as `AsMetadata`, return. If not found → return `OAuthError::DiscoveryFailed("no AS metadata found for {server_url}")`.

    Per D-07: 404 → try next fallback. 5xx → abort immediately.
    Per RESEARCH.md Pitfall 8: path is appended AFTER `.well-known/` prefix for RFC 9728.
    Use `tracing::debug!` to log each discovery step and URL tried.

    Tests: Use a local mock HTTP server (e.g., spin up an axum server on localhost in tests, or use `mockito` if available — check Cargo.toml). If neither is available, test URL construction logic via a helper `fn discovery_urls(server_url: &str) -> Vec<String>` that returns the ordered URLs to try, and unit test that function. Integration tests with real HTTP can use a simple tokio TCP server returning canned JSON.

    1. `pub async fn register_client(client: &reqwest::Client, registration_endpoint: &str, redirect_uri: &str) -> Result<DcrResponse, OAuthError>`:
       POST to `registration_endpoint` with JSON body:
       ```json
       {
         "client_name": "RightClaw",
         "redirect_uris": [redirect_uri],
         "grant_types": ["authorization_code"],
         "response_types": ["code"],
       ```

    2. `pub async fn register_client_or_fallback(client: &reqwest::Client, metadata: &AsMetadata, static_client_id: Option<&str>, redirect_uri: &str) -> Result<(String, Option<String>), OAuthError>`:
       - If `metadata.registration_endpoint` is Some → call `register_client`, return `(dcr.client_id, dcr.client_secret)`
       - If None and `static_client_id` is Some → return `(static_client_id.to_string(), None)`

       Per D-08: DCR fallback to static clientId from .mcp.json.

    3. `pub fn build_auth_url(metadata: &AsMetadata, client_id: &str, redirect_uri: &str, state: &str, code_challenge: &str, scope: Option<&str>) -> String`:
       Construct URL: `{metadata.authorization_endpoint}?response_type=code&client_id={}&redirect_uri={}&state={}&code_challenge={}&code_challenge_method=S256`
       If scope is Some, append `&scope={scope}`.
       URL-encode all parameter values.

    4. `pub async fn exchange_token(client: &reqwest::Client, token_endpoint: &str, code: &str, redirect_uri: &str, client_id: &str, client_secret: Option<&str>, code_verifier: &str) -> Result<TokenResponse, OAuthError>`:
       POST to `token_endpoint` with form-encoded body:
       ```
       grant_type=authorization_code
       code={code}
       redirect_uri={redirect_uri}
       client_id={client_id}
       code_verifier={code_verifier}
       ```
       If `client_secret` is Some, include `client_secret` in body.
       Content-Type: application/x-www-form-urlencoded
       Parse JSON response as `TokenResponse`. On non-2xx → `OAuthError::TokenExchangeFailed`.

    Tests: same approach as Task 1 — mock server or test URL/body construction helpers.
