**1. Create `crates/rightclaw/src/codegen/system_prompt.rs`** (per D-16, D-18):

```rust
use crate::agent::AgentDef;

/// Generate system prompt content for an agent.
///
/// If the agent has a `crons/` directory, the prompt instructs Claude
/// to run `/cronsync` on startup to reconcile scheduled tasks.
/// Returns `None` if no system prompt is needed.
pub fn generate_system_prompt(agent: &AgentDef) -> Option<String> {
    let crons_dir = agent.path.join("crons");
    if !crons_dir.is_dir() {
        return None;
    }

    Some(
        "## RightClaw System Instructions\n\n\
         On startup, check if the `crons/` directory exists in your agent directory.\n\
         If it contains `.yaml` files, run `/cronsync` to reconcile scheduled tasks.\n\n\
         This ensures all declared cron jobs are active after agent restart or session expiry.\n"
            .to_owned(),
    )
}
```

Returns `Option<String>` -- `None` when no system prompt needed (no crons/ dir). This is cleaner than returning an empty string.

   Before generating the shell wrapper for each agent, call `generate_system_prompt(agent)`. If it returns `Some(content)`:
   - Write content to `run_dir.join(format!("{}-system.md", agent.name))` (per D-16: `run/<agent>-system.md`)
   - Pass the path as `Some(path_str)` to `generate_wrapper`

2. **Updated loop** should look like:
```rust
for agent in &agents {
    // Generate system prompt if agent has crons/ directory (per D-16, D-21).
    let system_prompt_path = match rightclaw::codegen::generate_system_prompt(agent) {
        Some(content) => {
            let path = run_dir.join(format!("{}-system.md", agent.name));
            std::fs::write(&path, &content)
                .map_err(|e| miette::miette!(
                    "failed to write system prompt for '{}': {e:#}", agent.name
                ))?;
            tracing::debug!(agent = %agent.name, "wrote system prompt: {}", path.display());
            Some(path.display().to_string())
        }
        None => None,
    };

    let wrapper_content = rightclaw::codegen::generate_wrapper(
        agent,
        no_sandbox,
        system_prompt_path.as_deref(),
    )?;
    let wrapper_path = run_dir.join(format!("{}.sh", agent.name));
    std::fs::write(&wrapper_path, &wrapper_content)
        .map_err(|e| miette::miette!("failed to write wrapper for '{}': {e:#}", agent.name))?;
    std::fs::set_permissions(&wrapper_path, std::fs::Permissions::from_mode(0o755))
        .map_err(|e| miette::miette!(
            "failed to set wrapper permissions for '{}': {e:#}", agent.name
        ))?;
    tracing::debug!(agent = %agent.name, "wrote wrapper: {}", wrapper_path.display());
}
```

The system prompt file is regenerated on every `rightclaw up` invocation (per D-17: "regenerated on each `up`"), so there is no caching concern. The file lives in `run_dir` which is a temp directory per session.
