<objective>
Remove `--tunnel-hostname` CLI arg entirely. Derive the cloudflared public hostname
automatically from the JWT tunnel token: split on `.`, base64url-decode payload segment,
extract `"t"` field (tunnel UUID), return `<uuid>.cfargotunnel.com`.

Purpose: Operators no longer have to manually look up and copy their tunnel hostname.
The token already contains the UUID. One less arg, zero ambiguity.

<threat_model>
## Threat Model (ASVS L1)

**Input surface:** User-provided `--tunnel-token` string, and tokens stored in `config.yaml`.

| Threat | Input | Mitigation |
|--------|-------|------------|
| Wrong segment count | Token with 0, 1, 2, or 4+ dots | `split('.')` count check; fail with "expected 3 segments, got N" |
| Invalid base64url | Garbage in segment[1] | `base64::decode` returns `Err`; propagate via `?` with context |
| Invalid UTF-8 in decoded payload | Binary blob passed as token | `String::from_utf8` returns `Err`; propagate |
| Invalid JSON in payload | Valid base64 but not JSON | `serde_json::from_str` returns `Err`; propagate |
| Missing `"t"` field | Legitimate JWT but wrong issuer | Explicit field check; fail with "tunnel token payload missing 't' field" |
| Oversized payload | Attacker passes 1MB base64 blob | No heap amplification risk — base64 decode is proportional; serde_json parse is bounded by input; no network call |
| Empty string | `--tunnel-token ""` | Segment count check catches 0 dots → 1 segment → fails before decode |
| Null byte injection in UUID | `\0` in extracted `t` field | UUID is only used in `format!("{}.cfargotunnel.com", uuid)` — injected into a format string, not a shell command or SQL query. No shell exec at this call site. Benign. |

**Verdict:** All threats handled by fail-fast `?` propagation. No network calls, no exec, no filesystem writes during decode. Pure local computation.
</threat_model>

    ```rust
    use base64::Engine as _;

    impl TunnelConfig {
        pub fn hostname(&self) -> miette::Result<String> {
            let parts: Vec<&str> = self.token.split('.').collect();
            if parts.len() != 3 {
                return Err(miette::miette!(
                    "tunnel token has wrong number of segments (expected 3, got {})",
                    parts.len()
                ));
            }
            let payload_bytes = base64::engine::general_purpose::URL_SAFE_NO_PAD
                .decode(parts[1])
                .map_err(|e| miette::miette!("tunnel token payload is not valid base64url: {e:#}"))?;
            let payload_str = String::from_utf8(payload_bytes)
                .map_err(|e| miette::miette!("tunnel token payload is not valid UTF-8: {e:#}"))?;
            let payload: serde_json::Value = serde_json::from_str(&payload_str)
                .map_err(|e| miette::miette!("tunnel token payload is not valid JSON: {e:#}"))?;
            let uuid = payload["t"]
                .as_str()
                .ok_or_else(|| miette::miette!("tunnel token payload missing 't' field"))?;
            Ok(format!("{uuid}.cfargotunnel.com"))
        }
    }
    ```

    Note: `split('.')` (unbounded) correctly returns 4+ parts for tokens with extra dots, so the
    `parts.len() != 3` check rejects them with an accurate segment count.
