## What Was Built

### Task 1: rightclaw mcp status CLI

Added `McpCommands` enum with `Status { agent: Option<String> }` subcommand, `Commands::Mcp` variant, dispatch arm, and `cmd_mcp_status` function. The function iterates agent dirs (all or filtered), calls `mcp_auth_status` per agent, and prints a table with ✅/❌/⚠️ icons and AuthState Display labels. Returns "agent not found" error for unknown `--agent` filter. Tests cover: variant existence, agent-not-found error, no-.mcp.json success.

### Task 2: cloudflared --config + bot MCP warn

- **cloudflared-start.sh**: Added `cf_config_path_str = cf_config_path.display()` and inserted `--config {cf_config_path_str}` into the `exec cloudflared tunnel run` line so the generated ingress rules (with OAuth callback routes) are applied at tunnel startup.
- **bot startup warn**: Inserted `mcp_auth_status` check after `getMe` block, before cron spawn. Emits `tracing::warn!` with agent/server/state fields per server that is not `AuthState::Present`.

### Task 3: /mcp list icons + /doctor code block

- **handle_mcp_list**: Replaced `"OK"/"MISSING"/"EXPIRED"` literals with `"✅"/"❌"/"⚠️"` emoji and reformatted line as `"  {icon} {name}  —  {state}"` using `AuthState::Display`.
- **handle_doctor**: Separated body building from text construction; wraps body in `<pre>...</pre>` with `ParseMode::Html` — no escaping needed since doctor output is internally generated fixed-format text.
