**Gathered:** 2026-03-31

<domain>

`rightclaw up` writes `agent_dir/.claude/system-prompt.txt` per agent by composing the present

<decisions>
## Implementation Decisions

### system-prompt.txt Format

- **D-01:** `rightclaw up` writes `agent_dir/.claude/system-prompt.txt` using CC's `@file` reference
  syntax — one present identity file per line, in canonical order. Example output:
  ```
  @IDENTITY.md
  @SOUL.md
  @USER.md
  @AGENTS.md
  ```
  Paths are relative to agent_dir (which is the `cwd` for all `claude -p` calls).
  Absent files are silently skipped (no line written for them).

- **D-02:** File order must match OpenClaw convention. Preferred order: IDENTITY.md → SOUL.md →
  USER.md → AGENTS.md. **Researcher must verify this against OpenClaw spec before finalising.**

- **D-03:** `--system-prompt-file` flag used for `claude -p` bot invocations (Phase 25). Researcher
  must verify that CC's `--system-prompt-file` flag supports `@file` include syntax. If it does NOT,
  fall back to pre-concatenating file contents at generation time (raw content, `\n\n---\n\n` separator).

### Identity File Set

- **D-04:** All four OpenClaw identity files are candidates: IDENTITY.md, SOUL.md, USER.md, AGENTS.md.
  All present files included; absent files skipped without error. This matches OpenClaw convention
  and gives agents the full identity context CC needs.

- **D-05:** File-level write permissions must match OpenClaw conventions:
  - USER.md: agent-writable (user learns from interaction — expected to be updated by agent)
  - IDENTITY.md, SOUL.md, AGENTS.md: read-only from agent perspective (operator-defined)
  - system-prompt.txt: read-only (generated by `rightclaw up`, never edited by agent)
  **Researcher must confirm exact OpenClaw read/write semantics per file type and map to CC
  sandbox `allowWrite` entries in `settings.json`.**

### Hardcoded Sections Removed

- **D-06:** The old `generate_combined_prompt()` appended hardcoded sections (Startup Instructions,
  BOOTSTRAP.md detection, Communication reminder, Cron Management). These are ALL removed.
  No static content is appended after the identity file references.
  Agents that need this content must have it in their AGENTS.md or SOUL.md.

- **D-07:** BOOTSTRAP.md detection is gone from codegen. If an agent has BOOTSTRAP.md, the file
  is present in agent_dir and CC will see it via normal file discovery — no special handling needed
  in the system prompt generator.

### start_prompt Removed

- **D-08:** `start_prompt` field removed from `AgentConfig` entirely. No deprecation warning —
  clean break. If any existing agent.yaml files have this field, `deny_unknown_fields` on
  `AgentConfig` will cause a parse error, which is acceptable (fail-fast principle).

### Shell Wrapper Removal

- **D-09:** `codegen/shell_wrapper.rs` and its test file (`codegen/shell_wrapper_tests.rs`) are
  deleted. The template file `templates/agent-wrapper.sh.j2` is also deleted.
  `codegen/mod.rs` drops all `shell_wrapper` references and the `generate_wrapper` re-export.

- **D-10:** `generate_combined_prompt()` in `codegen/system_prompt.rs` is rewritten to produce
  the `@file` reference list (D-01) and renamed to `generate_system_prompt()`. Old prompt-file
  write logic in `cmd_up` is replaced with writing to `agent_dir/.claude/system-prompt.txt`.

- **D-11:** The old `run/<agent>-prompt.md` intermediate file is no longer written. The generated
  file lives at `agent_dir/.claude/system-prompt.txt` (alongside existing `settings.json`).

- **D-12:** Process-compose template (`templates/process-compose.yaml.j2`) and
  `codegen/process_compose.rs` are NOT modified in this phase. The `wrapper_path` field remains;
  PC config will be stale/broken until Phase 26.

### Default Right Agent Files

- **D-13:** Phase 24 creates SOUL.md, USER.md, and AGENTS.md for the default Right agent template
  (used by `rightclaw init`). Content:
  - SOUL.md: personality and values for the default "Right" agent
  - USER.md: empty or minimal placeholder (agent fills it in through interaction)
  - AGENTS.md: operational framework per Claude Code agent spec (replaces old hardcoded cron +
    communication sections that were in `generate_combined_prompt`)

### Claude's Discretion

- Exact content of default SOUL.md and AGENTS.md — Claude writes these following CC agent spec
  and OpenClaw conventions (see canonical refs below).
- Whether to keep `system_prompt_tests.rs` with new tests for the rewritten generator, or
  rewrite from scratch.
- Whether the function signature for `generate_system_prompt()` takes an `&AgentDef` or just
  `&Path` to the agent dir.

</decisions>
