<domain>
## Phase Boundary

Add `rightclaw pair <agent>` command that launches Claude Code interactively in the foreground — user can complete Telegram pairing and BOOTSTRAP.md onboarding. Process-compose captures stdin/stdout which blocks interactive TUI access.

</domain>

<decisions>
## Implementation Decisions

### Pair Command
- **D-01:** `rightclaw pair <agent>` launches Claude Code in foreground with the agent's combined prompt file
- **D-02:** Uses the same combined prompt from `run/<agent>-prompt.md` (generated by codegen)
- **D-03:** If prompt file doesn't exist, generate it first (same as `up` does)
- **D-04:** `rightclaw pair` without agent name defaults to "right"
- **D-05:** Pair command runs `claude` directly (no process-compose, no sandbox) — it's a setup tool, not production runtime
- **D-06:** Working directory is the agent's directory (same as `up`)

### Claude's Discretion
- Whether to regenerate the wrapper/prompt files or reuse existing ones
- Error handling when agent not found

</decisions>
