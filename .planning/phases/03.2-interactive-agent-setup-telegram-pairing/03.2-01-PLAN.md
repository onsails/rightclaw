Add a `Pair` variant to the `Commands` enum (per D-01):

```rust
/// Launch an agent interactively for setup (Telegram pairing, onboarding)
Pair {
    /// Agent name (defaults to "right")
    agent: Option<String>,
},
```

Wire it in the match block: `Commands::Pair { agent } => cmd_pair(&home, agent.as_deref())`.

Implement `cmd_pair(home: &Path, agent_name: Option<&str>) -> miette::Result<()>`:

1. Default agent name to "right" if None (per D-04).
2. Discover agents via `rightclaw::agent::discover_agents(&home.join("agents"))`.
3. Find the named agent in the list. If not found, return miette error listing available agents (same pattern as cmd_up's filter logic).
4. Create `run/` dir if missing: `std::fs::create_dir_all(&run_dir)?`.
5. Call `rightclaw::codegen::generate_combined_prompt(&agent)` to get prompt content (per D-02, D-03).
6. Write to `run/{agent_name}-prompt.md` (same path as cmd_up uses).
7. Resolve claude binary using `which::which("claude").or_else(|_| which::which("claude-bun"))`. Error if neither found, message: "claude CLI not found in PATH (tried: claude, claude-bun)".
8. Use `std::process::Command::new(claude_bin)` with args (per D-05, D-06, mirroring the no_sandbox branch of agent-wrapper.sh.j2):
   - `--append-system-prompt-file` pointing to the prompt file
   - `--dangerously-skip-permissions`
   - `-p` pointing to agent.path (working directory is agent's dir, per D-06)
9. Use `std::os::unix::process::CommandExt` `.exec()` to replace the current process (same pattern as cmd_attach). The call only returns on error, so wrap the return in `Err(miette::miette!("failed to launch claude: {err}"))`.

Do NOT add `--channels` flag -- pair is for interactive setup, not channel operation.
Do NOT add sandbox wrapping -- per D-05, pair is a setup tool.
