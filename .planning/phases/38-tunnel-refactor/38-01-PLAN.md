<objective>
Refactor `TunnelConfig` in `crates/rightclaw/src/config.rs` from JWT-token-based to credentials-file-based. Replace `{ token: String, hostname: String }` with `{ tunnel_uuid: String, credentials_file: PathBuf, hostname: String }`. Update serialization, deserialization, and backward-compat handling. Remove JWT decode method and base64 import.

Purpose: Establish the data model foundation that cmd_init (Plan 02) and the rest of Phase 38 depend on. All consumers read TunnelConfig from GlobalConfig — changing the struct here changes the contract for all downstream code.

Output: New `TunnelConfig` struct, updated `read_global_config` / `write_global_config`, backward-compat migration error for old `token:`-only configs, all tests updated and passing.
</objective>
