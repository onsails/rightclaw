<objective>
Write the DNS routing wrapper script at `~/.rightclaw/scripts/cloudflared-start.sh`, update process-compose template to use the script path as the cloudflared command, fix the MCP warning to use `eprintln!`, and update `cmd_up` to use `tunnel_cfg.hostname` field.

Purpose: The UUID-based cfargotunnel.com URL is not publicly routable without DNS. The wrapper script runs `cloudflared tunnel route dns <UUID> <HOSTNAME>` before starting the tunnel — making DNS routing automatic on each `rightclaw up`. `set -e` makes route failure fatal so process-compose restart behavior surfaces the problem.

Output: Updated cmd_up in main.rs, updated process_compose.rs + template, new script generation logic.

Depends on: Plan 01 (TunnelConfig.hostname field + tunnel_uuid() method must exist).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-06 | Tampering | `cloudflared-start.sh` contents | accept | Script written to operator-owned `~/.rightclaw/scripts/`; attacker with home-dir write access already has full operator control. Script is regenerated on every `rightclaw up`. |
| T-37-07 | Information Disclosure | Tunnel token embedded in shell script | accept | `~/.rightclaw/scripts/` is under operator's home dir (mode 700 by default). Token is already in `config.yaml` in the same directory tree. No new exposure surface. |
| T-37-08 | Elevation of Privilege | Script chmod 755 | accept | Script runs as the user who runs `rightclaw up`. No setuid, no privilege escalation path. |
| T-37-09 | Denial of Service | `cloudflared tunnel route dns` failure (set -e) | accept | Intentional by design per D-06: DNS routing failure is fatal to surfacing problems early. process-compose `restart: on_failure` handles retry. |
</threat_model>
