```rust
pub fn tunnel_uuid_from_credentials_file(path: &Path) -> miette::Result<String> {
    let content = std::fs::read_to_string(path)
        .map_err(|e| miette::miette!("read credentials file: {e:#}"))?;
    let v: serde_json::Value = serde_json::from_str(&content)
        .map_err(|e| miette::miette!("parse credentials file: {e:#}"))?;
    v["TunnelID"]
        .as_str()
        .map(|s| s.to_string())
        .ok_or_else(|| miette::miette!("credentials file missing 'TunnelID' field"))
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-supplied path → fs copy | `--tunnel-credentials-file` path comes from user CLI arg |
| copied file → cloudflared process | credentials file is passed to cloudflared via embedded config path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-04 | Tampering | credentials file copy at `~/.rightclaw/tunnel/<uuid>.json` | mitigate | Set 0600 permissions after copy — only owner can read/write. Verified via `set_permissions(mode(0o600))`. |
| T-38-05 | Information Disclosure | credentials file path in config.yaml | accept | config.yaml is in `~/.rightclaw/` — user-owned directory, no world read. Path disclosure is inert. |
| T-38-06 | Elevation of Privilege | `canonicalize()` on user-supplied credentials path | accept | canonicalize resolves symlinks but does not exec. Resulting path is passed to `std::fs::copy` only — no shell expansion, no exec. |
| T-38-07 | Denial of Service | route dns failure blocking cloudflared startup | mitigate | `|| true` makes route dns non-fatal — DNS record persists; cloudflared starts regardless. |
</threat_model>
