## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arg → filesystem probe | `--chrome-path` value from operator becomes PathBuf used in `detect_chrome_with_home()`; no shell expansion |
| which::which → PATH | MCP binary resolution via PATH; PATH is operator-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-01 | Spoofing | `--chrome-path` CLI arg | accept | Arg is `Option<PathBuf>` via clap — no shell expansion, no glob expansion. Operator provides their own path; they run as themselves. No privilege escalation vector. |
| T-43-02 | Tampering | `write_global_config` at end of cmd_init | accept | config.yaml is written to `~/.rightclaw/` which is operator-owned. No elevated privilege. File format is YAML with no executable semantics. |
| T-43-03 | Information Disclosure | `tracing::warn!` path output | accept | Logged paths are paths the operator configured or that exist on their system. No secrets in path strings. Severity: low. |
| T-43-04 | Elevation of Privilege | `brew --prefix` subprocess call | accept | `brew` is a user-space tool; invoked via `Command::new("brew")` with no shell, no argument injection (no user input reaches brew call). macOS-only. Returns one line of text parsed as PathBuf. |
</threat_model>
