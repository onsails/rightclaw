<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env → process | RC_RIGHTCLAW_HOME env var read at MCP server startup — must not expose sensitive path injection |
| .mcp.json write | generate_mcp_config writes RC_RIGHTCLAW_HOME into agent's .mcp.json — value comes from trusted rightclaw up caller |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | RC_RIGHTCLAW_HOME env var | accept | Value is a file path read from env; used only to construct PathBuf passed to read_global_config. No code execution. Worst case: wrong config path → mcp_auth returns error. Low risk. |
| T-01-02 | Spoofing | RC_RIGHTCLAW_HOME path traversal | accept | Path is used only to read config.yaml via read_global_config(). If attacker controls env, they already control the process. No privilege escalation possible. |
| T-01-03 | Information Disclosure | RC_RIGHTCLAW_HOME in .mcp.json | accept | .mcp.json is owned by the agent user; home dir path is not a secret. Acceptable disclosure. |
</threat_model>
