Implementation details per locked decisions:
- **D-01**: `sandbox.enabled: !no_sandbox`, `autoAllowBashIfSandboxed: true`, `allowUnsandboxedCommands: false`
- **D-02**: `sandbox.filesystem.allowWrite` starts with `agent.path.display().to_string()` (absolute path, NOT `~/` prefix -- see anti-pattern in research)
- **D-03**: `sandbox.network.allowedDomains` defaults: `["api.anthropic.com", "github.com", "npmjs.org", "crates.io", "agentskills.io", "api.telegram.org"]`
- **D-04**: Top-level: `skipDangerousModePermissionPrompt: true`, `spinnerTipsEnabled: false`, `prefersReducedMotion: true`
- **D-05**: Conditional `enabledPlugins` when `agent.mcp_config_path.is_some()`
- **D-08/D-06**: Merge user overrides via `Vec::extend()` for all three array fields
- **D-12/D-13**: `no_sandbox` only changes `sandbox.enabled` to `false`, everything else identical
- Security defaults (Claude's discretion from research): `sandbox.filesystem.denyRead: ["~/.ssh", "~/.aws", "~/.gnupg"]`
- Omit `excludedCommands` from JSON when empty (cleaner output)
