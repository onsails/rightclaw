<objective>
Harden sandbox path resolution to use absolute paths and add allowRead support, fixing the critical bug where denyRead tilde paths resolve to agent HOME under override (making denyRead ineffective).

Purpose: Under HOME override, `~/.ssh` resolves to `$AGENT_DIR/.ssh` (which doesn't exist) instead of the real host `~/.ssh`. This makes denyRead security controls completely ineffective. Additionally, agents need explicit `allowRead` for their own directory (which may be inside the denied host HOME path).

Output: `generate_settings()` uses absolute host HOME paths for denyRead, includes allowRead with agent path, `SandboxOverrides` has new `allow_read` field, all callers updated, all tests updated.
</objective>
