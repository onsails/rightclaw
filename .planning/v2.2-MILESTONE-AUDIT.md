### 2. No .claude/skills/skills/ stale cleanup in cmd_up (D-10)

**Decision:** Phase 14 explicitly deferred adding stale `skills/` dir cleanup to `cmd_up` because
the project is not yet in production — no real agents have the intermediate `skills/` install.

**Impact:** When shipped, agents upgraded from a Phase 12 state (which installed to `.claude/skills/skills/`)
will retain a stale `skills/` directory. This will not cause errors (rightskills/ will still be installed
correctly) but leaves cruft.

**Action required before v2.3 release:**
```rust
// In cmd_up agent loop, add before install_builtin_skills():
let _ = std::fs::remove_dir_all(agent.path.join(".claude/skills/skills"));
```
This mirrors the existing SKILLS-05 pattern for clawhub cleanup.
