## E2E Flow Verification

All 4 E2E flows verified by integration checker against actual codebase.

### Flow 1: Telegram message → CC dispatch → response
`dispatch.rs` → `filter_map(chat_id)` → `handle_message()` → `effective_thread_id()` → DashMap → `spawn_worker()` → debounce → `invoke_cc()` → `get_session()`/`create_session()` → `claude -p --agent --session-id` or `--resume` → `parse_reply_output()` → `split_message()` → `bot.send_message()`

### Flow 2: /reset → new session
`handle_reset()` → `worker_map.remove()` + `delete_session()` → next message → `create_session()` with new UUID

### Flow 3: Cron job execution
`run_cron_task()` → `reconcile_jobs()` → `run_job_loop()` → `execute_job()` → `is_lock_fresh()` guard → `INSERT cron_runs status=running` → `claude -p --agent <name>` → `UPDATE cron_runs exit_code/status` → `remove_file(lock)`

### Flow 4: rightclaw up → bot process
`cmd_up()` → `generate_agent_definition()` (`.claude/agents/<name>.md`) → `generate_process_compose()` (`<name>-bot: rightclaw bot --agent`) → `process-compose up`
