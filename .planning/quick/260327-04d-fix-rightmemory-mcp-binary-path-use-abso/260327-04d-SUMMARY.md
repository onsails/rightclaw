# Quick Task 260327-04d: Fix rightmemory MCP binary path — use absolute path

**One-liner:** `generate_mcp_config` now accepts `binary: &Path` and writes it verbatim to `.mcp.json` `command` field; caller passes `current_exe()` so agents find rightclaw even when it is not on PATH.

## What Was Done

`generate_mcp_config` previously hardcoded `"command": "rightclaw"` in `.mcp.json`. When process-compose spawns agents, the sandbox environment does not inherit the user's PATH, so `rightclaw` could not be found and the rightmemory MCP connection failed.

Fix:
- Signature changed to `generate_mcp_config(agent_path: &Path, binary: &Path)`
- `binary.to_string_lossy()` replaces the hardcoded string
- `crates/rightclaw-cli/src/main.rs`: resolves `std::env::current_exe()` once before the per-agent loop, passes result to each `generate_mcp_config` call
