# Milestone v2.2: Skills Registry

**Status:** SHIPPED 2026-03-26

## Overview

Replace ClawHub with skills.sh as the primary skill registry, ship `/rightskills` skill manager, add per-agent env var injection via `agent.yaml`, rework the policy gate for CC-native sandbox, and clean up all v2.2 tech debt.

## Phases

### Phase 11: Env Var Injection

**Goal**: Users can declare per-agent env vars in agent.yaml that are safely injected into the agent's shell environment on every `rightclaw up`
**Depends on**: v2.1 complete (Phase 10)
**Plans**: 2 plans

Plans:

- [x] 11-01: env field in AgentConfig, shell wrapper injection, single-quote escaping, 6 new tests (ENV-01, ENV-02, ENV-03)
- [x] 11-02: installed.json create-if-absent fix, agent.yaml template comment with plaintext warning (ENV-04, ENV-05)

**Requirements covered:** ENV-01, ENV-02, ENV-03, ENV-04, ENV-05

**Key decisions:**
- Single-quote escaping via `replace('\'', "'\\''")` — safe for $, backticks, spaces without shell expansion
- `env_exports` built as `Vec<String>` in Rust before template rendering — pre-formatted export lines
- Positioned between ANTHROPIC_API_KEY capture and `export HOME=` (D-03)
- `installed.json` create-if-absent follows Phase 9 `settings.local.json` pattern exactly
