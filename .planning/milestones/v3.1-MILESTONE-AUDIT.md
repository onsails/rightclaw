### Phase 31: verify-sandbox.sh VER-01 description overclaim
- **Item:** Stage 4 summary output (lines 238-239) states it confirms "same claude binary + settings.json as teloxide worker" — this is accurate for the bot subprocess initial call but not for Telegram worker resume calls, which use `--resume` and omit `--agent`.
- **Impact:** Documentation accuracy only. Sandbox engagement is correctly proven by exit code under `failIfUnavailable:true` regardless of this difference.
- **Recommended action:** Add comment in script noting the `--agent`-always behavior matches cron.rs exactly; Telegram worker first-call is also covered. Future v3.2 work on `--resume` path could add a separate smoke test.

### Phase 29: SBOX-04 unverifiable statically
- **Item:** `devenv.nix pkgs.ripgrep` only verifiable at runtime (live nix devenv shell). Static code analysis cannot confirm `rg` is in PATH.
- **Impact:** None in practice — the fallback chain (absent rg → `rg_path=None` → settings.json omits key → `failIfUnavailable:true` aborts CC → DOC-02 warns before launch) is intact and tested.
- **Recommended action:** Document in onboarding that `direnv allow` / `nix develop` must be active before `rightclaw up`.
